# Fruit Tab Cards

Fruit Tab Cards 是一个 Chrome 标签页管理插件。它会把当前窗口里的标签页变成卡片，帮你快速看出哪些页面占内存、哪些很久没看、哪些是重复打开的页面，并支持拖拽整理标签组。

适合经常一口气打开很多标签页的人使用。

## 你可以用它做什么

- 查看当前窗口里所有标签页
- 按“建议关闭”“重复”“已收藏”快速筛选标签页
- 一键关闭插件建议清理的标签页
- 找出重复打开的网页，并保留其中一个
- 收藏或取消收藏网页
- 拖拽标签页到已有标签组
- 拖拽标签页新建标签组
- 修改标签组名称和颜色
- 从“回收站”恢复通过插件关闭的标签页

## 安装方法

这个插件目前是本地开发版，需要手动加载到 Chrome。

1. 打开 Chrome。
2. 在地址栏输入 `chrome://extensions/` 并回车。
3. 打开右上角的“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择这个项目文件夹：`tab-card-plugin`。
6. 加载成功后，工具栏会出现 Fruit Tab Cards 的插件图标。

以后如果你修改了代码，需要回到 `chrome://extensions/`，点击这个插件卡片上的“重新加载”按钮。

## 第一次使用

点击 Chrome 工具栏里的 Fruit Tab Cards 图标，插件会打开一个独立的标签页管理页面。

如果管理页面已经打开，再次点击插件图标会跳回已有页面，不会重复打开多个管理页。

页面主要分成两块：

- 左侧：总览、搜索、Tab Groups、回收站
- 右侧：当前选中范围内的标签页卡片

## 左侧怎么用

### 搜索

在搜索框里输入标题、网址、域名或标签组名称，可以快速找到相关标签页。

### 总览数字

- 总内存：插件能估算到的标签页内存总量
- 建议关闭：插件认为可以清理的标签页数量
- 重复 Tab：重复打开的标签页副本数量
- 标签组：当前窗口里的 Chrome 标签组数量

### Tab Groups

这里展示浏览器当前窗口的整体标签情况。

- 全部标签：当前窗口下所有标签页
- 未分组：没有加入 Chrome 标签组的标签页
- 其他项目：已有的 Chrome 标签组

点击某个分组后，右侧只显示这个分组里的标签页。左侧的数量始终表示整体情况，不会因为右侧筛选而变化。

### 新建标签组

你可以把右侧任意标签页卡片拖到左侧“拖到这里新建组”区域，插件会为它创建一个新标签组。

### 回收站

通过插件关闭的标签页会暂时放在回收站里。点开回收站后，可以恢复之前关闭的页面。

## 右侧怎么用

### 筛选

右侧顶部有几个筛选按钮：

- 全部：显示当前左侧选中范围内的所有标签页
- 建议关闭：显示当前范围内建议清理的标签页
- 重复：显示当前范围内重复打开的标签页副本
- 已收藏：显示当前范围内已经收藏的网页

这些筛选只作用于左侧当前选中的分组。例如你在左侧点了“未分组”，右侧的“建议关闭”就只表示未分组里的建议关闭项。

### 卡片

每张卡片代表一个 Chrome 标签页。

卡片上会显示：

- 页面标题
- 页面链接或域名
- 内存参考值
- 上次访问时间
- 是否为当前页面
- 是否建议关闭
- 是否重复
- 是否已收藏

点击卡片可以跳转到对应标签页。

### 收藏

点击卡片上的星标按钮，可以收藏或取消收藏网页。

只有普通 `http://` 和 `https://` 网页支持收藏。`chrome://`、扩展页、本地文件等页面可能无法收藏。

### 关闭标签页

点击卡片上的关闭按钮，可以关闭这个标签页。被插件关闭的标签页会进入左侧回收站。

### 关闭建议项

点击右上方的“关闭建议项”，会批量关闭当前插件判断为建议清理的标签页。

建议先看一眼列表，确认没有需要保留的页面。

## 标签组操作

你可以直接在页面里整理 Chrome 标签组：

- 拖拽卡片到某个分组：把标签页移动到该组
- 拖拽卡片到“未分组”：把标签页移出标签组
- 修改分组名称：点击分组标题输入新名称
- 修改分组颜色：使用分组右侧的颜色选择
- 取消分组：把组内标签页移出分组
- 删除分组：关闭该组内的标签页

## 数据说明

### 上次访问时间

卡片里的“打开”使用 Chrome 提供的 `lastAccessed` 数据，含义更接近 Chrome 标签搜索里看到的“多久前访问过”。

点击刷新按钮会重新读取一次 Chrome 标签页数据。插件页面打开后，也会每 10 分钟自动刷新一次。

### 内存

Chrome 稳定版没有稳定开放精确的单标签页内存 API。插件会尽量读取可用的进程内存信息；如果拿不到，会尝试读取页面 JS heap 作为参考。

有些页面因为浏览器限制无法读取内存，会显示为 `--`。这是正常情况。

### 建议关闭

插件会把以下标签页标记为建议关闭：

- 重复打开的标签页副本
- 已被 Chrome 丢弃或冻结的标签页
- 长时间未访问的标签页
- 内存占用较高的标签页

同一个链接重复打开时，插件会保留 1 个标签页，只把其他副本标记为重复和建议关闭。优先保留当前正在看的页面；如果都不是当前页面，则保留最近访问的那个。

## 常见问题

### 为什么有些页面内存显示为 `--`？

浏览器不允许扩展读取某些页面的数据，比如 `chrome://` 页面、扩展页面、本地文件或权限受限页面。这种情况下插件会显示 `--`。

### 为什么刚加载插件时数据有变化？

插件页面打开后会读取当前 Chrome 窗口的实时标签页数据。刷新、关闭、恢复、拖拽分组都会让数据重新计算。

### 会不会把标签页真的删掉？

点击单个卡片关闭按钮或“关闭建议项”时，会真的关闭 Chrome 标签页。但通过插件关闭的页面会先进入插件回收站，可以在回收站里恢复。

### 这个插件会上传我的浏览记录吗？

不会。插件在本地读取 Chrome 当前窗口的标签页信息，用于展示和管理；没有远程服务器，也不会上传你的浏览记录。

## 开发说明

这是一个 Manifest V3 Chrome 插件，主要文件包括：

- `manifest.json`：插件配置
- `background.js`：插件后台逻辑
- `popup.html`：管理页面结构
- `popup.css`：页面样式
- `popup.js`：标签页数据读取、筛选、拖拽和交互逻辑

本项目不需要安装依赖，直接通过 Chrome 的“加载已解压的扩展程序”加载即可。
